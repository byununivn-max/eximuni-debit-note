# 메뉴 구조 개편 (menu-restructure) - 계획서

> 작성일: 2026-02-12
> 상태: Plan (사용자 검토 대기)

---

## 1. 배경 및 목적

EXIMUNI ERP의 사이드바 메뉴를 업계 표준(CargoWise, SAP TM, NewageNXT 등) 물류/포워딩 ERP 구조에 맞춰 재구성한다.

---

## 2. 현재 메뉴 구조

```
EXIMUNI ERP (현재)
├─ 대시보드 (6개 하위)
│  ├─ 매출/매입 현황
│  ├─ P&L 손익
│  ├─ 수익성 분석
│  ├─ 고객 수익성
│  ├─ 건별 수익성
│  └─ 견적-실적
├─ 운영 (4개)
│  ├─ 거래 데이터
│  ├─ CD 통관
│  ├─ Ops 운영
│  └─ CO 원산지
├─ 매입 (2개)
│  ├─ 공급사
│  └─ 매입 관리
├─ 매출 (2개)
│  ├─ 매출 종합
│  └─ Debit Note
├─ 회계 (9개)
│  ├─ 계정과목
│  ├─ 분개전표
│  ├─ 시산표
│  ├─ 재무제표
│  ├─ AP 공급사
│  ├─ AR 고객
│  ├─ 비용 분류
│  ├─ 월별 비용
│  └─ SB 임포트
├─ 마스터데이터 (3개)
│  ├─ 거래처 관리
│  ├─ 회계기간
│  └─ 환율 관리
└─ 관리자 (1개)
   └─ 감사 이력
```

**총 7개 그룹, 27개 하위 메뉴**

---

## 3. 현재 구조의 문제점

| 문제 | 상세 |
|------|------|
| **대시보드 과부하** | 6개 분석 메뉴가 대시보드에 집중 (홈과 분석 보고서가 혼재) |
| **회계 과부하** | 9개 하위 메뉴 (AP/AR 보조원장 + 원가 분석 + 임포트가 섞임) |
| **매입/매출 빈약** | 각 2개씩으로 독립 그룹 가치 낮음 |
| **업무 흐름 불일치** | 운영→정산→회계 흐름이 메뉴에 반영되지 않음 |

## 4. 업계 표준 조사 요약

### 조사 대상 ERP

- **CargoWise** (WiseTech Global) - 업계 1위, 216개+ 모듈
- **SAP Transportation Management** (SAP TM)
- **NewageNXT** (Newage Global)
- **Logi-Sys** (Softlink Global)
- **Shipthis**
- **DP World CARGOES Runner**
- **GoFreight**
- **Magaya Supply Chain**

### 업계 공통 모듈 분류 패턴 (6~8개 대분류)

```
1. Sales & CRM (영업)
   - 리드/문의 관리, 견적(Quotation) 생성/발송, 고객 관리
   - 영업 목표/활동 추적, 요율/관세표(Tariff) 관리

2. Operations (운영)
   - Shipment/Job 생성 및 관리, 부킹(Booking) 관리
   - 상태 추적(Tracking), 컨테이너/화물 관리, 스케줄 관리

3. Documentation (서류)
   - B/L (MBL/HBL), AWB (MAWB/HAWB), Delivery Order
   - Cargo Arrival Notice, 포워딩 지시서, 매니페스트

4. Customs & Compliance (통관/컴플라이언스)
   - 수입/수출 통관 신고, HS코드 분류, 원산지 증명
   - 규정 준수(스크리닝), 전자 신고(EDI/ABI)

5. Finance & Accounting (재무/회계)
   - 인보이싱/빌링, AP (매입) / AR (매출), GL (총계정원장)
   - 다통화 처리, 세금/VAT 관리, Debit/Credit Note, 현금 흐름

6. Reports & Analytics (보고서/분석)
   - 대시보드(KPI), 수익성 분석(고객별/무역노선별/영업사원별)
   - P&L 리포트, 미결 잔액 리포트(Aging), 경영 정보 시스템(MIS)

7. Master Data (마스터 데이터)
   - 거래처(고객/공급사/에이전트), 환율 관리
   - 항구/공항/도시 코드, 요율표(Rate Sheet), 회계 기간/계정과목

8. Administration (관리)
   - 사용자/권한 관리, 감사 이력, 시스템 설정, 워크플로우 설정
```

### 사용자 업무 흐름

```
견적(Quotation) → 부킹(Booking) → 운영(Operation/Job) → 서류(Documentation) → 통관(Customs) → 정산(Billing) → 회계(Accounting) → 분석(Analytics)
```

### ERP 네비게이션 UX 원칙

1. **명사형 라벨 사용**: "운영", "회계", "보고서" 등
2. **계층은 최대 2단계**: 대분류 > 하위 메뉴
3. **논리적 그룹핑**: 관련 기능끼리 시각적으로 구분
4. **일관된 아이콘**: 대분류마다 직관적인 아이콘
5. **상단 고정 메뉴**: 대시보드(홈)는 항상 최상단, 설정/관리는 최하단

## 5. 제안 - 안 A: 업무 흐름 중심 (Operations-First)

> 포워딩 업무 흐름 순서(운영→매매→회계→분석)를 메뉴 배치에 반영

```
EXIMUNI ERP
│
├─ 📊 홈                              ← 축소 (종합 현황 1개)
│     매출/매입 현황 (기존 DashboardPage)
│
├─ 🚢 운영 (Operations)               ← 핵심 업무
│     거래 데이터 (Shipments)
│     CD 통관
│     Ops 운영
│     CO 원산지
│
├─ 💰 매매 관리 (Trading)              ← 매입+매출 통합
│     매출 종합
│     Debit Note
│     공급사
│     매입 관리
│
├─ 📒 회계 (Accounting)               ← 핵심 회계만
│     계정과목
│     분개전표
│     시산표
│     재무제표
│     AP 공급사 (보조원장)
│     AR 고객 (보조원장)
│     SB 임포트
│
├─ 📈 분석/보고서 (Analytics)          ← 대시보드에서 분리
│     P&L 손익
│     수익성 분석
│     고객 수익성 (GP 랭킹)
│     건별 수익성
│     견적-실적 비교
│     비용 분류
│     월별 비용 집계
│
├─ 🗄️ 마스터 데이터                    ← 유지
│     거래처 관리
│     회계기간
│     환율 관리
│
└─ ⚙️ 관리자
      감사 이력
```

**장점**: 업무 흐름에 충실, 매매 통합으로 직관적, 분석 메뉴 독립
**단점**: 매입/매출 구분이 약해질 수 있음

---

## 6. 제안 - 안 B: 역할 기반 (Role-Based)

> 사용자 역할(운영팀/재무팀/경영진)에 따라 그룹핑

```
EXIMUNI ERP
│
├─ 📊 대시보드                         ← 경영진용 2개만
│     종합 현황
│     P&L 손익
│
├─ 🚢 운영                            ← 운영팀
│     거래 데이터 (Shipments)
│     CD 통관
│     Ops 운영
│     CO 원산지
│
├─ 🛒 매입                            ← 매입 담당
│     공급사
│     매입 관리
│
├─ 💵 매출/정산                        ← 매출+정산 담당
│     매출 종합
│     Debit Note
│     견적-실적 비교
│
├─ 📒 회계                            ← 회계팀 (핵심 장부)
│     계정과목
│     분개전표
│     시산표
│     재무제표
│     SB 임포트
│
├─ 🏦 AP/AR 관리                      ← 채권채무 전담 분리
│     AP 공급사 (보조원장)
│     AR 고객 (보조원장)
│
├─ 📈 경영 분석                        ← 경영진/관리자
│     수익성 분석
│     고객 수익성 (GP 랭킹)
│     건별 수익성
│     비용 분류
│     월별 비용 집계
│
├─ 🗄️ 마스터 데이터
│     거래처 관리
│     회계기간
│     환율 관리
│
└─ ⚙️ 관리자
      감사 이력
```

**장점**: 역할별 접근 명확, 매입/매출 구분 유지, AP/AR 독립으로 채권채무 업무 집중
**단점**: 그룹 수가 9개로 다소 많음

## 7. 비교 요약
